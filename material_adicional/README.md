### 8.5 Segurança

Para segurança de contratos, o mínimo recomendado é combinar bibliotecas auditadas, testes automatizados, revisão manual e ferramentas de análise estática.

Também vale introduzir desde cedo o hábito de pensar em permissões, upgrades, controle de acesso, validações, reentrância, tratamento de erros, simulação de transações e observabilidade.

**Ferramentas/temas úteis para acrescentar na trilha:**
- OpenZeppelin Contracts e guias de segurança.
- Foundry para testes mais rápidos e fuzzing básico do ecossistema.
- Hardhat para workflows populares de deploy e integração.
- Slither e Echidna para análise estática e fuzz testing em Solidity.
- Tenderly para debugging e tracing.

### 10.1 Onde IA ajuda de verdade

IA ajuda bastante em onboarding, explicação de conceitos, geração de testes, revisão inicial de contratos, documentação de funções, scaffolding de projetos e transformação de código entre frameworks — mas não substitui auditoria nem validação manual.

Em blockchain, o melhor uso de IA é como copiloto técnico: pedir explicações passo a passo, checklist de segurança, exemplos mínimos reprodutíveis e tradução de docs complexas para linguagem mais acessível.

### 10.3 Prompts úteis

**Prompt 1 — Explicação beginner friendly**

```text
Explique o conceito de [tema] em blockchain como se a pessoa soubesse programação básica, mas nunca tivesse visto Web3. Use analogias simples, depois conecte com a implementação real em [Bitcoin/Ethereum/Solana/Stellar].
```

**Prompt 2 — Leitura guiada de documentação**

```text
Resuma esta página de documentação em português claro. Separe em: o que é, quando usar, pré-requisitos, passos práticos, erros comuns e glossário mínimo.
```

**Prompt 3 — Geração de exercício**

```text
Crie 3 exercícios progressivos sobre [tema], um iniciante, um intermediário e um avançado, com objetivo, passos esperados, dicas e critérios de correção.
```

**Prompt 4 — Segurança de contratos**

```text
Revise este contrato inteligente com foco em: controle de acesso, validação de inputs, reentrância, pausabilidade, upgrades, eventos, testes necessários e riscos de produção. Não assuma que o código está correto.
```

**Prompt 5 — Pair programming blockchain**

```text
Atue como pair programmer para construir um projeto em [rede]. Primeiro proponha a arquitetura, depois gere estrutura de pastas, dependências, contratos/programas, testes, scripts de deploy e checklist de segurança. Explique cada decisão.
```

### 10.4 Skills para alunos

**Skills de IA recomendadas para desenvolvimento em blockchain:**
- Saber pedir explicação em camadas: visão simples, visão técnica e visão de produção.
- Saber colar contexto suficiente: versão da linguagem, framework, rede, objetivo e erro encontrado.
- Saber pedir testes antes de pedir deploy.
- Saber pedir análise de ameaça e não apenas "corrija o código".
- Saber validar toda resposta contra docs oficiais e ambiente local/testnet.

**Boas práticas de uso de IA:**
- Sempre informar stack exata: Solidity/Foundry, Solidity/Hardhat, Rust/Anchor, Soroban, Bitcoin Core etc.
- Pedir respostas em formato operacional: passo a passo, comandos CLI, arquivos criados, testes e riscos.
- Solicitar referências para a doc oficial correspondente.
